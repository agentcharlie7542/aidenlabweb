CHAPTER 01

프로젝트 개요

현행 사이트는 imweb 기반의 브랜드 카탈로그 형태로, 회사의 핵심 경쟁력인 "글로벌 마케팅·커머스 운영 실력"과 "わさび10이라는 자체 플랫폼"을 충분히 표현하지 못하고 있다. 이번 리뉴얼은 신뢰감 있는 기업 사이트 + 리드 전환 머신 + wasabi10 게이트웨이라는 세 가지 역할을 동시에 수행하도록 설계한다.

1.1 비즈니스 목표 (Why)

PRIMARY GOAL

신규 B2B 리드 증대

월 30건 → 월 150건의 무료 진단 신청. 폼 작성률 12% 이상, 1영업일 내 응답 100%.

SECONDARY

wasabi10 자사몰 유입

전체 세션의 18% 이상을 wasabi10 신규 회원가입까지 전환. UTM·이벤트로 추적.

SECONDARY

채용·팀빌딩 강화

사이트를 통한 자발적 지원 월 20건 → 월 60건. 임직원 평균 추천 점수 4.3 이상.

  • 스크롤 진입 페이드 업: 모든 섹션은 viewport 진입 시 12px 아래에서 0.4초 페이드. Intersection Observer 1회 트리거.

7.3 모바일·반응형

  • 모바일 최우선 설계 (320px부터 보장). 폼·CTA는 sticky bottom bar로 항상 노출.

9.2 i18n 라우팅

  • 기본 한국어 — prefix 없음 (/about)
  • 영어 — /en/about, 일본어 — /ja/about
  • 최초 접속 시 Accept-Language + IP 기반 자동 라우팅, 사용자 선택 시 쿠키 저장
  • HTML lang + hreflang 자동 삽입 (Sanity 다국어 문서 매핑)
  • 현재 분리된 aidenlab-us.com / aidenlab-jp.com 도메인은 301 리다이렉트로 통합

9.3 wasabi10 연동

  • 외부 링크: 모든 wasabi10 링크는 rel="noopener noreferrer" + UTM 자동 부여
  • 이벤트 트래킹: GA4 click_wasabi10_outbound 커스텀 이벤트, GTM 트리거
  • 단일 SSO (옵션): 향후 aidenlab.io 로그인 = wasabi10 계정 통합 가능성을 두고 NextAuth + 공유 세션 설계 검토
  • 크로스 도메인 분석: GA4 cross-domain measurement 활성화로 동일 사용자 추적

9.4 인프라·보안

  • HTTPS 강제, HSTS preload
  • Sanity Studio 접근은 SSO + IP 화이트리스트
  • 폼 제출 시 reCAPTCHA v3 + 허니팟 필드
  • 월 1회 의존성 패치, Lighthouse CI를 GitHub Actions로 자동화
